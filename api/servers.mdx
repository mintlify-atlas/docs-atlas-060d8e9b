## Test Server Connection

<RequestExample>
```bash cURL
curl -X POST http://localhost:3000/api/servers/test \
  -H "Content-Type: application/json" \
  -d '{
    "baseUrl": "https://192.168.1.100",
    "apiKey": "your-api-key-here",
    "trustSelfSigned": true
  }'
```
</RequestExample>

Tests connectivity to an Unraid server before adding it.

<ParamField body="baseUrl" type="string" required>
  The base URL of the Unraid server (http or https). Must not include credentials in the URL.
</ParamField>

<ParamField body="apiKey" type="string" required>
  The API key for authenticating with the Unraid server. Maximum length: 4096 characters.
</ParamField>

<ParamField body="trustSelfSigned" type="boolean" default="true">
  Whether to trust self-signed SSL certificates.
</ParamField>

<ResponseField name="success" type="object">
  Returns connection test results including server information and available scopes.
</ResponseField>

**Error Responses:**
- `400` - Invalid request body or malformed baseUrl
- `502` - Connection test failed (server unreachable or authentication failed)

## Create Server

<RequestExample>
```bash cURL
curl -X POST http://localhost:3000/api/servers \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: your-csrf-token" \
  -H "Cookie: unpwa_csrf=your-csrf-token" \
  -d '{
    "baseUrl": "https://192.168.1.100",
    "apiKey": "your-api-key-here",
    "requestedScopes": ["read", "write"],
    "name": "My Unraid Server",
    "accentColor": "#3b82f6",
    "trustSelfSigned": true
  }'
```
</RequestExample>

Creates a new server connection and stores its credentials securely.

**Authentication:** Requires CSRF token

**Rate Limit:** 20 requests per minute per IP

<ParamField body="baseUrl" type="string" required>
  The base URL of the Unraid server (http or https).
</ParamField>

<ParamField body="apiKey" type="string" required>
  The API key for authenticating with the Unraid server. Maximum length: 4096 characters.
</ParamField>

<ParamField body="requestedScopes" type="string[]" default="[]">
  Array of requested permission scopes. Each scope max length: 128 characters.
</ParamField>

<ParamField body="name" type="string">
  Custom name for the server. If empty or not provided, will auto-resolve from server. Maximum length: 120 characters.
</ParamField>

<ParamField body="accentColor" type="string">
  Accent color for the server (hex color like #3b82f6 or legacy values: amber, orange, purple, blue, green). Maximum length: 32 characters.
</ParamField>

<ParamField body="trustSelfSigned" type="boolean" default="true">
  Whether to trust self-signed SSL certificates.
</ParamField>

<ResponseField name="ok" type="boolean">
  Indicates successful server creation.
</ResponseField>

<ResponseField name="serverId" type="string">
  The unique identifier for the created server.
</ResponseField>

<ResponseField name="activated" type="boolean">
  Whether the server was automatically set as the active server.
</ResponseField>

**Error Responses:**
- `400` - Invalid request body or validation errors
- `429` - Rate limit exceeded
- `500` - Unable to store server credentials

## List Servers

<RequestExample>
```bash cURL
curl http://localhost:3000/api/servers
```
</RequestExample>

Retrieves all configured server connections (without API keys).

<ResponseField name="activeServerId" type="string | null">
  The ID of the currently active server, or null if none is active.
</ResponseField>

<ResponseField name="servers" type="array">
  Array of server configurations.
  <Expandable title="Server object">
    <ResponseField name="id" type="string">
      Unique server identifier.
    </ResponseField>
    <ResponseField name="name" type="string">
      Server display name.
    </ResponseField>
    <ResponseField name="accentColor" type="string">
      Server accent color.
    </ResponseField>
    <ResponseField name="baseUrl" type="string">
      Server base URL.
    </ResponseField>
    <ResponseField name="trustSelfSigned" type="boolean">
      Whether self-signed certificates are trusted.
    </ResponseField>
    <ResponseField name="scopes" type="string[]">
      Configured permission scopes.
    </ResponseField>
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of server creation.
    </ResponseField>
  </Expandable>
</ResponseField>

**Error Responses:**
- `500` - Unable to list servers

## Update Server

<RequestExample>
```bash cURL
curl -X PUT http://localhost:3000/api/servers/{serverId} \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: your-csrf-token" \
  -H "Cookie: unpwa_csrf=your-csrf-token" \
  -d '{
    "name": "Updated Server Name",
    "accentColor": "#10b981",
    "trustSelfSigned": false
  }'
```
</RequestExample>

Updates an existing server configuration.

**Authentication:** Requires CSRF token

**Rate Limit:** 20 requests per minute per IP

<ParamField path="id" type="string" required>
  The server ID to update. Maximum length: 160 characters.
</ParamField>

<ParamField body="name" type="string">
  Updated server name. If empty string provided, will auto-resolve from server. Maximum length: 120 characters.
</ParamField>

<ParamField body="trustSelfSigned" type="boolean">
  Whether to trust self-signed SSL certificates.
</ParamField>

<ParamField body="accentColor" type="string">
  Updated accent color (hex or legacy color name). Maximum length: 32 characters.
</ParamField>

<ParamField body="apiKey" type="string">
  Updated API key. Maximum length: 4096 characters.
</ParamField>

<Note>
At least one field (name, trustSelfSigned, accentColor, or apiKey) must be provided.
</Note>

<ResponseField name="ok" type="boolean">
  Indicates successful update.
</ResponseField>

**Error Responses:**
- `400` - Invalid request body or validation errors
- `404` - Server not found
- `429` - Rate limit exceeded
- `500` - Unable to update server

## Test API Key

<RequestExample>
```bash cURL
curl -X POST http://localhost:3000/api/servers/{serverId}/test-key \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: your-csrf-token" \
  -H "Cookie: unpwa_csrf=your-csrf-token" \
  -d '{
    "apiKey": "new-api-key-to-test"
  }'
```
</RequestExample>

Tests a new API key against an existing server configuration.

**Authentication:** Requires CSRF token

**Rate Limit:** 20 requests per minute per IP

<ParamField path="id" type="string" required>
  The server ID to test against.
</ParamField>

<ParamField body="apiKey" type="string" required>
  The API key to test. Maximum length: 4096 characters.
</ParamField>

<ResponseField name="success" type="object">
  Returns connection test results with the new API key.
</ResponseField>

**Error Responses:**
- `400` - Invalid request body
- `404` - Server not found
- `429` - Rate limit exceeded
- `502` - API key test failed

## Get Server Status

<RequestExample>
```bash cURL
curl http://localhost:3000/api/servers/status
```
</RequestExample>

Retrieves the currently active server configuration and connection status.

<ResponseField name="configured" type="boolean">
  Whether a server is configured and active.
</ResponseField>

<ResponseField name="id" type="string">
  Active server ID (only present if configured is true).
</ResponseField>

<ResponseField name="name" type="string">
  Server name (only present if configured is true).
</ResponseField>

<ResponseField name="accentColor" type="string">
  Server accent color (only present if configured is true).
</ResponseField>

<ResponseField name="baseUrl" type="string">
  Server base URL (only present if configured is true).
</ResponseField>

<ResponseField name="trustSelfSigned" type="boolean">
  Whether self-signed certificates are trusted (only present if configured is true).
</ResponseField>

<ResponseField name="scopes" type="string[]">
  Configured permission scopes (only present if configured is true).
</ResponseField>

<ResponseField name="canWrite" type="boolean">
  Whether the current configuration has write permissions (only present if configured is true).
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of server creation (only present if configured is true).
</ResponseField>

**Error Responses:**
- `500` - Unable to load server status
